Blog is now powered by Hexo

2018/04/15

I’ve been using gatsbyjs and faced a lot of issues on user defined components. Adding asciinema never worked. It took me days to cleanly render my Terminal component. Most of the pluggins were still under development. Rendering a pdf was really a nightmare which at last was successful but not in a clear way. The gatsby pdf has never been started. The themes were lacking. The RSS plugin didn’t work at all. The problems with definition files in typescript mode was huge. The webpack bundle size hell was real even when I’ve tried plugging uglify modules.

Not to say I really liked React Helmet which is just like native react Helmet. The styled components, typography, KaTex, GraphQL and mainly React were fabolous.

The site is now powered by Hexo, another static site generator. Hexo does provide cool plugins and every plugin worked. But not hexo-pdf. I’m really on to it now. The coolest thing I’ve found was the asciinema plugin. And also, not to mention, hexo uses PHP in the background.
